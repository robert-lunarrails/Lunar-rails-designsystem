09 — AI Rules

Do's and Don'ts for AI

Rules the AI must follow when building any Lunar Rails product, page, or component.

✅ DO
  • Use --gradient-brand on hero text, CTAs, and stat numbers
  • Use --color-bg-page (#0A0A0A) as base background — always dark
  • Use Satoshi for ALL text — sole brand typeface across headings, body, and UI
  • Add backdrop-filter: blur() on nav bar and modals
  • Use fadeUp with staggered delays on section and card entries
  • Apply --shadow-glow-purple on hover states for cards and buttons
  • Use border: 1px solid var(--color-border-default) on all cards
  • Keep copy bold and direct: "Launch" · "Build" · "Trace" · "Analyse"
  • Align all spacing to the 8px token scale
❌ DON'T
  • Use light or white backgrounds — dark-theme-only brand
  • Use Montserrat, Inter, Roboto, Arial — Satoshi is the only brand typeface
  • Apply gradient to body text, captions, or anything below --text-h3
  • Use purple gradients on white/light surfaces (classic AI cliché)
  • Use glassmorphism as primary UI pattern — nav only
  • Hard-code hex values — always use CSS tokens
  • Use more than 2 accent colors per section (outside gradient elements)
  • Add drop shadows to text
  • Use border-radius above --radius-2xl (24px) on non-pill elements
  • Mix neon and pastel colors — palette is dark + vivid, never soft
